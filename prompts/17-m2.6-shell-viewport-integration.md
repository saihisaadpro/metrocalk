<deliverables>
1. **The single-window shell (ADR-008).** Transparent WebView2 (the M2.5 React UI) over a native wgpu surface on one HWND, OS-composited — the path M2.3 confirmed — plus M2.3's input-routing layer for the real hand-off. No DComp/CEF/self-composite. If anything regresses M2.3's clean composite at integration, **stop and reconcile against ADR-008** rather than working around it.
2. **Viewport renders the real scene.** Wire M2.2's instanced path (entity meshes + gizmos + grid; GPU frustum cull → indirect draws) to the live scene from the real `/core`; camera + pixel-picking in Rust (never crossing JS).
3. **Real core + real transport — no mocks.** Swap M2.5's `MockCore` for the WASM/Rust `/core` (commit pipeline + registry + `World`); replace the editor's stubbed desktop transport binding with the real Tauri Channel (M2.4); peer-identity handshake on connect.
4. **The round-trip.** Select in viewport → selection delta → M2.5 inspector → edit field / bind target → JSON-Patch `EditTx` → `/core` commit pipeline → delta echoes over M2.4 → viewport + UI update; Ctrl-Z via the engine-side undo stack (M1.6). The 60 Hz drag stays in Rust; only the commit delta crosses (invariants 2 & 4).
5. **Browser parity.** The wasm32 build renders the scene (Loro-authoritative per ADR-006); document any desktop/browser divergence.
6. **Close the M2 deferred residuals** (each lane parked these *for here* — measure, don't assume):
   - **M2.3 shell:** the **DPI 100%↔200% monitor move**, a **min-spec / truly low-end** machine, and a **sustained (uninterrupted ≥60 s) human flicker watch** under motion (M2.3 only sampled ~10 s).
   - **M2.4 transport:** the **real Tauri Channel end-to-end latency** on Windows (re-confirm M2.1's ~3.4 ms over the real shell) and the **Windows initial-snapshot-load cliff** (route startup via WS/fragments if it breaches budget).
   - **M2.5 editor:** re-measure the **O(n)-per-delta store apply in a real browser** (jsdom over-reported ~tens of ms at 5k); if committed-edit latency at scale is a problem, apply structural sharing (persistent map → ~O(1)/entity).
7. **The M2-exit assessment.** Does the integrated editor hold the frame budget on the real scene? Quote the end-to-end numbers (render + commit + delta + UI update). This is the M2 go/no-go.
</deliverables>
