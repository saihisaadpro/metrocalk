<task>
Decide how the editor shell composites the React UI over the native wgpu viewport on Windows, and write **ADR-008**.

**New evidence — this changes the premise.** M2.1's automated 1b reported the single-window approach blacking out / collapsing to 16×16. But the spike was explicit that (a) **GDI cannot capture a GPU overlay swapchain**, so the "all black" could be a *capture artifact*, and (b) a live human look was the gold-standard verdict. **That human look has now happened: a triangle rendered in the composited window and dragging was responsive — no blackout, interactive.** That strongly suggests the catastrophic "FAIL" was a **capture/automation artifact, not a real compositing failure**, and that the **cheapest possible shell — plain single-window Tauri + transparent WebView2 + wgpu — may actually be viable**.

So **retire the cheapest path first**: rigorously confirm or refute single-window against the FULL gate battery (the conditions neither GDI nor a quick drag exercised). Fall to **DComp/child-webview** only if a specific gate condition fails; fall to **CEF self-composite** only if DComp also fails. Do not pay the DComp/CEF cost (HAL-level DX12 / ~170 MB) until single-window is genuinely disproven.
</task>

<scope>
In scope (`/spikes/shell-composite`, Windows): (1) the path-agnostic input-routing layer; (2) the single-window composite with the **real React panel layout** + the full battery; (3) the DComp variant *only if* single-window fails a gate; (4) confirming the CEF fallback is reachable *only if* DComp fails; (5) the decision → ADR-008 + a status-note revising ADR-007's 1b conclusion. Out of scope: the real editor/viewport (M2.6), the transport (M2.4).
</scope>

<deliverables>
1. **Input-routing layer (first, path-agnostic).** Per-pixel hit test: pointer over UI → webview; over viewport → native wgpu picking/gizmo; handles partial overlap (a dropdown over the viewport). Carries unchanged to any shell, so it's never wasted.
2. **Single-window confirmation (the cheap path, tried first).** The **real** React panel layout — dock + toolbar + a floating inspector/dropdown that *overlaps* the viewport — as a transparent-region WebView2 over a wgpu surface on one HWND. Run the full battery (below). **Capture with a method that actually sees the swapchain** (camera/capture card, or disciplined human observation against the checklist) — NOT GDI, which is exactly what misled M2.1.
3. **Fallback ladder (only on a real single-window failure).** DComp / `CreateCoreWebView2CompositionController` (WebView2 Visual hosting — wry doesn't expose it) + wgpu **DX12 swapchain-from-visual** (HAL-level, no safe wgpu-29 API — gfx-rs/wgpu #2434) in one `IDCompositionVisual` tree. If DComp also fails → confirm CEF self-composite (`tauri-apps/cef-rs`; offscreen UI → shared D3D11 texture → composite in the wgpu pass; Graphite's proven path) is reachable.
4. **Decision → ADR-008**, plus a **status-note on ADR-007** recording the outcome honestly: if single-window passes, 1b's "FAIL" was a GDI capture artifact (state it); if it reproduces interactively, confirm the real failure and proceed down the ladder.
</deliverables>

<success_criteria>
The FULL battery, **applied to single-window first**, on **≥2 GPUs incl. an Intel iGPU** (M2.1's blackout was driver-sensitive):
- **Real UI composited** — the actual React panels render over the viewport, transparent regions show the viewport through them, correct z-order, **no blackout/collapse** under sustained interactive use. (The human pass already covered responsiveness + no-blackout for a triangle; this adds the real panels + everything below.)
- **No flicker under sustained motion** — ≥60 s continuous viewport animation with the UI on screen (Tauri #9220 territory).
- **No flicker/desync under resize + DPI** — rapid resize + a 100%↔200% monitor move: layers stay aligned, text crisp.
- **Correct input routing under partial overlap** — a dropdown/toast over the viewport: events hit the right layer, nothing falls through.
- **60 Hz frame budget** holds; no added drag latency.
**Pass all ⇒ single-window IS the shell** (simplest — no DComp, no CEF, no ~170 MB): ADR-008 = single-window, ADR-007 revised. **Fail any ⇒ DComp; fail DComp ⇒ CEF.**
</success_criteria>
