# Prompt 15 — M2.4: Transport + binary delta protocol (`/transport`)

> Use with `00-orchestrator.md` (v2). **Effort `xhigh`.** Grounded in M2.1's IPC verdict (Channel 3.4 ms / WS 1.3 ms p99, overhead-bound). Independent of the shell-composition spike and M2.2.
> **Lane:** Owns `/core`'s producer hook + the `/transport` crate (trait + protocol + impls) · Blocked by nothing (M2.1 evidence is in) · **Parallel-safe with M2.3 (shell spike)** · feeds M2.5 (the UI consumes this contract) · see `prompts/README.md`.
> Read first: ADR-002 (Loro), ADR-006 (browser is Loro-authoritative), ADR-007 (the IPC verdict), invariants 2 & 4, and the research reality below.

---

<task>
Build the deltas-only transport layer (invariant 2): the wire **protocol**, the backend-agnostic `DeltaTransport` trait, its three impls, and the `/core` producer hook that turns each committed transaction into a coalesced delta on the wire.
Research-verified design (June 2026): **adopt the Loro Syncing Protocol v1 framing** (4 magic bytes + room id + message-type byte + payload; `DocUpdate` batches N update blobs + an 8-byte batch id for ACK; fragment header/fragment for >256 KB; explicit `proto_version`; app-level ping/pong) and carry **Loro's own native `export({mode:"update", from: vv})` bytes as the opaque payload** — do NOT invent a custom per-op delta codec, and do NOT ship bare Loro bytes without the envelope. This reuses Loro's version-vector causality so ordering, idempotency, and out-of-order tolerance come for free, and the same wire serves the Phase-2 collab path unchanged.
</task>

<scope>
In scope: the protocol framing (envelope + Loro-update payload + coalescing + fragments + versioning); a byte-oriented `DeltaTransport` trait with the envelope/ACK/backpressure/reconciliation logic in a shared layer ABOVE it; three impls — Tauri Channel (desktop default), local WebSocket (fallback + collab foundation), in-process WASM (browser, zero-copy); the `/core` producer hook (`subscribe_local_update` → frame-tick coalescer → send); Wait-for-Ack reconciliation; a reserved `%EPH` ephemeral frame-kind. Out of scope: the editor UI (M2.5), the actual collab server (Phase 2), CEF/DComp shell wiring (M2.6). `/transport` must NOT link Loro — payloads are `&[u8]` (keep the CI `loro`-outside-`/core` grep green).
</scope>

<deliverables>
1. **Protocol.** Adopt the Loro Protocol v1 frame layout (reuse the MIT Rust impl from `github.com/loro-dev/protocol` where it fits, or port just the framing into `/transport`). Envelope: `proto_version`, magic `frame_kind` (`%LOR` document delta · `%EPH` ephemeral — reserve it now even if unused in M2), room id, payload-len. Payload = Loro `update` bytes. Coalesce a frame's worth of 60 Hz deltas into one `DocUpdate`; support fragmentation for the initial catch-up/snapshot.
2. **`DeltaTransport` trait** — `send(frame: &[u8])`, `set_on_recv(cb)`, `connection_state()`. Envelope encode/decode, batch-id/ACK, the **outbox-collapse backpressure** (if the UI falls behind, advance the `from` version and send ONE update spanning the gap — never queue N frames), and reconciliation live in a shared module above the trait, written once.
3. **Three impls:** (a) **Tauri Channel** (desktop default; `Channel<InvokeResponseBody>` raw bytes, NOT JSON; re-confirm M2.1's 3.4 ms p99 on the stress scene, and separately benchmark the **initial snapshot load over Channel on Windows** — the WebView2 large-payload cliff — routing startup via WS/fragments if it fails); (b) **local WebSocket** (M2.1: 1.3 ms; doubles as the collab foundation); (c) **in-process WASM** (zero-copy `Uint8Array` view over wasm linear memory; degenerates to a sync callback since the browser is Loro-authoritative).
4. **`/core` producer hook** — on commit-pipeline completion, `subscribe_local_update` → a single frame-tick coalescer (`from = last_sent_version`) → enveloped frame to the active transport. Establish **peer identity in the handshake** (set Loro PeerID per ADR-002 F3) before any delta flows.
5. **Wait-for-Ack reconciliation** (M2 single-process: core acks instantly) with a per-mutation "unacknowledged" marker, **structured so the full undo→apply→redo server-reconciliation is a drop-in upgrade** when collab brings concurrent remote ops. Keep this reconciliation-undo strictly separate from the engine-side inverse-op *user* undo stack (M1.6/ADR-002 F2).
</deliverables>

<success_criteria>
The protocol round-trips Loro updates losslessly; `/transport` links no `loro`/Flecs types (CI grep green); all three impls send/recv enveloped frames; the producer hook emits one coalesced delta per tick; backpressure collapses to one update spanning the gap (not N queued); Wait-for-Ack clears the unacknowledged marker on echo; the `%EPH` frame-kind is reserved. Desktop Channel re-confirms ≤~4 ms p99 on the stress scene; the Windows snapshot-load path is measured (and routed to WS/fragments if it breaches budget).
</success_criteria>

<verification>
Tests (seeded): out-of-order import convergence (apply update2 before update1 → `pending` then convergence); idempotent re-import (no-op); reconnect resync from a stale version vector; **backpressure-collapse** (stall the sink, assert one update spanning the gap, not N frames); fragment timeout/resend; optimistic-echo no-op (predicted == authoritative) vs correction (validation repaired an invariant). Bench the Channel path twice per discipline. Adversarial paragraph: "the case coupling the wire to Loro's format bites us" — and why the opaque-payload envelope contains it.
</verification>
