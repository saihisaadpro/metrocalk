# Prompt 09 — M1.5: 16 ms compatibility-query CI perf gate

> Use with `00-orchestrator.md` (v2) as system prompt. **Effort `high`.** Run **after M1.4** (needs the shared 5k fixture) and M1.2 (the wrapper query).
> Read first: M1.4's scene presets, spike ②'s latency table (`spikes/flecs/README.md`), and the orchestrator's benchmark discipline. This closes M1's "foundation" arc.

---

<task>
Wire the compatibility query into CI as a hard performance gate, so the product's core interactivity promise (north-star test #1, `<16 ms`) can never silently regress. After this session, a change that pushes the cached compat query over budget fails the build — the same tripwire discipline spike ③ used for wasm.
</task>

<scope>
In scope: a criterion (or equivalent) bench running the M1.2 wrapper's cached compatibility query on M1.4's 5k preset; a CI job that fails above threshold; the threshold calibrated to the CI runner (not a dev machine) with a recorded margin.
Out of scope: new query features, scene changes, the commit pipeline. Just the gate.
</scope>

<deliverables>
1. A reproducible bench of the cached compat query on the 5k preset, **through the wrapper** (not raw Flecs).
2. A CI job that **FAILS if p99 > 16 ms** on the runner. Current is ~12 µs, so the gate has ~1000× headroom — that is intentional: it catches order-of-magnitude regressions (an accidental uncached path, a fragmentation blowup), not noise.
3. A **calibration note**: the runner's measured baseline + the chosen margin, documented so future maintainers know why the threshold sits where it does (runner hardware ≠ the i9 the spikes used).
</deliverables>

<success_criteria>
CI fails on a >16 ms p99 query (prove it: temporarily inject a slow/uncached path, watch CI go red, revert) · passes at the current ~12 µs · threshold calibrated to the runner with a recorded margin · the bench reuses M1.4's fixture, not a bespoke scene.
</success_criteria>

<verification>
Demonstrate the gate BOTH green (normal) and red (deliberate slow injection, then reverted) — exactly the proof spike ③'s wasm-tripwire required. Re-run twice; confirm the threshold isn't flaky against runner noise (the huge headroom should make it rock-solid — if it's flaky, the bench is measuring the wrong thing).
</verification>

<definition_of_done>
☐ bench on the 5k preset through the wrapper · ☐ CI gate green normally, verified red on a deliberate regression, then reverted · ☐ threshold calibrated to runner + margin documented · ☐ progress.md updated — M1 foundation (M1.1–M1.5) complete; Now points at **M1–2 (ECS↔Loro commit pipeline + merge-validation + engine-side undo stack)** + dated log entry · ☐ architecture.md CI/perf-gate note · ☐ working tree clean, committed.
</definition_of_done>
