<task>
Fix the robustness gaps the M1 audit surfaced in the commit pipeline + undo stack, each with a regression test that would have caught it. The pipeline is invariant 3 ‚Äî editor, plugins, and AI all mutate through it ‚Äî so its correctness and atomicity are load-bearing for everything after M1.
</task>

<scope>
In scope (`/core`): precise additive-SetField undo; atomic commit; reverse `tid‚Üíeid` index; error propagation on the mutation path; the missing regression tests. Out of scope: merge/collab capability rebuild (Phase-2 ‚Äî see the DoD note), new features, any M2 work.
</scope>

<deliverables>
1. **Fix the undo over-removal bug (real correctness defect).** Today the inverse of `SetField` on a *newly-added* field (`old_value = None`) is `RemoveComponent`, which deletes the **whole** component. So undoing "add field B to a component that already held field A *from a prior transaction*" destroys A. The in-tree comment calls this "conservative but correct" ‚Äî it is **not** correct across transactions. Add `Op::RemoveField { entity, component, field }` and make the additive-`SetField` inverse remove **only that field**. Regression test: tx1 sets `Health.hp`; tx2 sets `Health.maxHp`; **undo tx2 ‚áí `Health.hp` survives**, `Health.maxHp` gone. (Today this test fails ‚Äî Health is wholly removed.)
2. **Atomic commit (the claimed-but-unenforced guarantee).** `commit` applies ops one-by-one with no rollback and does **not** use `World::defer` despite the docstring saying "via defer." A mid-transaction failure ‚Äî e.g. an op referencing a stale entity, exactly the plugin/AI case invariant 3 invites ‚Äî leaves a partial mutation **and** pushes no undo entry (un-revertable). **Pre-validate every op before applying any** (entity/parent existence, etc.); reject the whole transaction on any invalid op so it is all-or-nothing. Correct the docstring. Test: a 3-op tx whose 2nd op is invalid ‚áí `Err` **and zero state change**.
3. **Reverse `tid‚Üíeid` index.** `tid_to_eid` is an O(n) linear scan called inside `collect_subtree`'s BFS and per-subtree-entity during delete-inverse capture ‚Üí O(subtree √ó scene) on a large-subtree delete. Maintain a `tid ‚Üí EntityId` map alongside `eid_to_tid`.
4. **Error propagation on the mutation path.** Replace `.unwrap()` on fallible Loro ops in `apply_*` with `PipelineError::Loro` propagation (the variant already exists). Keep undo/redo failures loud, but document why.
</deliverables>

<success_criteria>

</success_criteria>
