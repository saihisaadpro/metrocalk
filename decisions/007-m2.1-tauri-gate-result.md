# ADR-007: M2.1 exit-gate result — Tauri IPC confirmed, single-window compositing fails → self-composite

**Date:** 2026-06-14 · **Status:** Accepted · **Resolves:** the M2 exit gate deferred by
[ADR-003](003-desktop-first-tauri-exit-gate.md) ("Revisit when: M2 gate — 60 Hz interaction on
Windows WebView2"). · **Evidence:** `spikes/tauri-shell/` (throwaway), `spikes/tauri-shell/evidence/`.

## Context

ADR-003 chose Tauri 2 desktop-first but gated the keep-Tauri-vs-CEF decision on a Windows WebView2
benchmark, framed around an IPC bandwidth floor (~200 ms / 10 MB) and Graphite's abandonment of Tauri.
The M2.1 spike resolved it with **two co-equal sub-gates** on real Windows WebView2
(**149.0.4022.69**, Tauri 2.11.2 / wry 0.55.1, wgpu 29.0.3 Vulkan, RTX 4060 laptop).

## Findings

**1a — IPC delta wire: PASS.** Real Loro coalesced deltas (`core::Engine`, 60 Hz transform drag) over
three raw-binary transports, 2×60 s / 3600 frames each:

| transport (RTT) | p99 | dropped @ 60 Hz |
|---|---|---|
| Tauri Channel (engine→UI push) | **3.4–3.6 ms** | 0 |
| local WebSocket (CEF-style) | **1.3–1.7 ms** | 0–1 |
| `invoke` raw-binary (UI→engine) | 6.7–6.8 ms | 0–2 |

The real delta is **103 bytes** — ~10,000× under the measured ~31.8 MB/s ceiling. The wire is
**overhead-bound, not bandwidth-bound**: ADR-003's "~200 ms / 10 MB" is a large-payload bandwidth
figure that never applies to a deltas-only protocol. The streaming transports (Channel, WebSocket)
that carry 60 Hz render-deltas pass with multiple-× headroom; the request/response `invoke` path has a
**~4 ms fixed per-call overhead** and should not be the 60 Hz channel.

**1b — compositing: FAIL on the direct single-window approach.** A transparent WebView2 *does*
composite on Windows (control run: desktop shows through the transparent viewport). But attaching a
native **wgpu surface to the same window HWND** blacks out the window and destabilizes it — the
webview UI that renders fine on its own disappears. This reproduces, empirically and on this exact
stack, the conflict Graphite hit ([Graphite #2541](https://github.com/GraphiteEditor/Graphite/issues/2541),
[Tauri #9220](https://github.com/tauri-apps/tauri/issues/9220),
[#11944](https://github.com/tauri-apps/tauri/discussions/11944)). Not yet tested: the Tauri `unstable`
child-webview split / DComp visual layering; and an eyes-on human visual pass (the harness runs with
one command for it).

## Decision

1. **IPC leg of ADR-003 is CONFIRMED** — and its bandwidth framing corrected to "overhead-bound,
   small deltas." Use **Channel / WebSocket** for the 60 Hz delta stream; reserve `invoke` for
   episodic UI→engine commits.
2. **Compositing leg:** the naive "transparent webview over a native wgpu viewport in one Tauri
   window" **does not work on Windows** via a single-HWND surface. Plan the shell around a
   **self-composite** path — offscreen-render / hand the UI layer into the wgpu pass (Graphite's
   proven approach) — rather than relying on OS webview transparency over a native surface.
3. **Before fully pivoting to a CEF wrapper,** run a small follow-up spike on the Tauri `unstable`
   child-webview / DComp-layering variant: if it composites cleanly, plain Tauri stays viable (IPC is
   fine, transparency works); if not, the CEF self-composite shell is the path. Verso stays
   "revisit-when-mature."

Net: **Tauri is not disqualified** (IPC passes, transparency works), but the viewport↔UI composition
must be self-managed — do not assume the one-window overlay.

## Consequences

- The renderer must own the UI↔viewport composition (UI-as-texture in the wgpu pass), not delegate it
  to OS webview transparency over a native HWND surface.
- M2 build's transport choice is unblocked: deltas-only over Channel/WebSocket is proven on Windows.
- A child-webview/DComp follow-up spike is the cheap option to keep a plain Tauri shell; budget it
  before committing to CEF.

## Revisit when

The child-webview/DComp variant is tested; a min-spec machine is measured (current numbers are dev-box
with large headroom but argued, not measured, for min-spec); or Tauri ships compositor-friendly
webview embedding.
