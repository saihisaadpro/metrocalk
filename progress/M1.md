- **`metrocalk-ecs` crate stood up — the engine's relational query surface.** The `World` trait is anchored to the product's compatibility queries (ADR-006), not Flecs's feature set: pair-match `(Provides,X)`, wildcard `(R,*)`, negation, read-target (`targets`/`for_each_edge`), plus the minimal entity/pair/tag mutations + `defer`. Resolved the spike's `with`/`without` ambiguity by exposing ONE form (a `Clause` over ids → value-form `with((rel,target))`, wildcard via `flecs::Wildcard::ID`). Safety locks ON (default features).
