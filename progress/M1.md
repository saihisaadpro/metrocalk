This is the honest call: a real memory win that isn't worth its current costs yet. (Storage model unchanged тЖТ no architecture.md change.)

### 2026-06-13 (M1.3 тАФ component metadata registry)
- **`metrocalk-core::registry` built тАФ the data layer behind the compatibility query.** `Registry<W: World>` (generic over the M1.2 backend) interns each capability as an `Entity` and records every registered kind as an entity carrying `(Provides, cap)` / `(Requires, cap)` / `(Observes, cap)` pairs, so `providers_of("Health")` / `requirers_of` / `observers_of` answer **through the wrapper's pair-match query** тАФ no reaching behind it, `flecs_ecs` stays out of `core` (CI grep stays green).
- **JSON-Schema registration + validation:** `ComponentMeta::builder(..).fields_from_json_str(schema)` parses the component-field subset (object/properties/scalar types/required/format/descriptionтЖТui-hint) and rejects malformed schemas with **actionable, non-panicking** `RegistryError`s (root-not-object, unsupported-type naming the field, missing-type, required-unknown-field, invalid-JSON, тАж). Builder chosen over a derive macro (less magic; same runtime path a plugin/marketplace component uses).
- **Serialization round-trip:** `to_json`/`from_json`; fields canonicalized (sorted), `serde(skip)` on empties тЖТ all 12 stdlib components serialize тЖТ deserialize тЖТ re-serialize **byte-identical**. `stdlib::standard_components()` = 12 real kinds (Transform, Health, HealthBar, Sprite, MeshRenderer, RigidBody, Collider, AudioSource, Light, Camera, Animator, Script) with a real capability web (Sprite+MeshRenderer both provide `Renderable`; HealthBar requires+observes `Health`).
- Tests (6 green): capability queries through the wrapper (`providers_of("Renderable") == [MeshRenderer, Sprite]`, `requirers_of("Health") == [HealthBar]`), byte-identical round-trip per component, JSON-schema parse (types/required/format/hints), malformed-schema rejection (6 cases, each actionable), empty-name + duplicate rejection. Workspace clippy `-D warnings` + fmt clean.
- **ADR-004 marketplace gap (verification):** a marketplace component registers through this exact `from_json`/`fields_from_json_str` path (it's metadata-as-JSON). Gaps Phase 2 must close: **(1) capability namespacing** тАФ two authors' `"Health"` capability currently collide on the same interned id; need canonical/namespaced capability ids. **(2)** the registry holds *metadata only* тАФ a marketplace component also ships assets + behavior (WASM), which are separate Phase-2 systems (asset pipeline, Extism host). **(3)** schema **versioning/migration** (plan ┬з5.7) тАФ `ComponentMeta` needs a version + migration path before external components persist.

### 2026-06-13 (M1.2 тАФ ECS wrapper API + Flecs backend)
- **`metrocalk-ecs` crate stood up тАФ the engine's relational query surface.** The `World` trait is anchored to the product's compatibility queries (ADR-006), not Flecs's feature set: pair-match `(Provides,X)`, wildcard `(R,*)`, negation, read-target (`targets`/`for_each_edge`), plus the minimal entity/pair/tag mutations + `defer`. Flecs-only powers (transitive, `$var` joins, hierarchy) are deliberately **not** exposed. Resolved the spike's `with`/`without` ambiguity by exposing ONE form (a `Clause` over ids тЖТ value-form `with((rel,target))`, wildcard via `flecs::Wildcard::ID`). Safety locks ON (default features).
- **Cross-check:** spike тСб's compat query expressed through the trait returns **exactly 211** matches @5k (== independently-tracked truth == spike number), 1999 edges; `for_each_edge` and `targets` agree on edge count; deferred mutation applies. 3 integration tests green.
