### 2026-06-13 (M1.3 — component metadata registry)
- **`metrocalk-core::registry` built — the data layer behind the compatibility query.** `Registry<W: World>` (generic over the M1.2 backend) interns each capability as an `Entity` and records every registered kind as an entity carrying `(Provides, cap)` / `(Requires, cap)` / `(Observes, cap)` pairs, so `providers_of("Health")` / `requirers_of` / `observers_of` answer **through the wrapper's pair-match query** — no reaching behind it, `flecs_ecs` stays out of `core` (CI grep stays green).
- **JSON-Schema registration + validation:** `ComponentMeta::builder(..).fields_from_json_str(schema)` parses the component-field subset (object/properties/scalar types/required/format/description→ui-hint) and rejects malformed schemas with **actionable, non-panicking** `RegistryError`s (root-not-object, unsupported-type naming the field, missing-type, required-unknown-field, invalid-JSON, …). Builder chosen over a derive macro (less magic; same runtime path a plugin/marketplace component uses).
- **Serialization round-trip:** `to_json`/`from_json`; fields canonicalized (sorted), `serde(skip)` on empties → all 12 stdlib components serialize → deserialize → re-serialize **byte-identical**. `stdlib::standard_components()` = 12 real kinds (Transform, Health, HealthBar, Sprite, MeshRenderer, RigidBody, Collider, AudioSource, Light, Camera, Animator, Script) with a real capability web (Sprite+MeshRenderer both provide `Renderable`; HealthBar requires+observes `Health`).
- Tests (6 green): capability queries through the wrapper (`providers_of("Renderable") == [MeshRenderer, Sprite]`, `requirers_of("Health") == [HealthBar]`), byte-identical round-trip per component, JSON-schema parse (types/required/format/hints), malformed-schema rejection (6 cases, each actionable), empty-name + duplicate rejection. Workspace clippy `-D warnings` + fmt clean.
- **ADR-004 marketplace gap (verification):** a marketplace component registers through this exact `from_json`/`fields_from_json_str` path (it's metadata-as-JSON). Gaps Phase 2 must close: **(1) capability namespacing** — two authors' `"Health"` capability currently collide on the same interned id; need canonical/namespaced capability ids. **(2)** the registry holds *metadata only* — a marketplace component also ships assets + behavior (WASM), which are separate Phase-2 systems (asset pipeline, Extism host). **(3)** schema **versioning/migration** (plan §5.7) — `ComponentMeta` needs a version + migration path before external components persist.

### 2026-06-13 (M1.2 — ECS wrapper API + Flecs backend)
- **`metrocalk-ecs` crate stood up — the engine's relational query surface.** The `World` trait is anchored to the product's compatibility queries (ADR-006), not Flecs's feature set: pair-match `(Provides,X)`, wildcard `(R,*)`, negation, read-target (`targets`/`for_each_edge`), plus the minimal entity/pair/tag mutations + `defer`. Flecs-only powers (transitive, `$var` joins, hierarchy) are deliberately **not** exposed. Types are our own opaque `Entity(u64)` / `Target` / `Term` / `Clause` — no `flecs_ecs` type in any trait signature.
- **Native Flecs backend** (`FlecsWorld`) behind the trait; everything is a runtime id (no `#[derive(Component)]`), matching how the registry (M1.3) will hand out ids and what a Loro backend can offer. Resolved the spike's `with`/`without` ambiguity by exposing ONE form (a `Clause` over ids → value-form `with((rel,target))`, wildcard via `flecs::Wildcard::ID`). Safety locks ON (default features).
- **Cross-check:** spike ②'s compat query expressed through the trait returns **exactly 211** matches @5k (== independently-tracked truth == spike number), 1999 edges; `for_each_edge` and `targets` agree on edge count; deferred mutation applies. 3 integration tests green.
- **Zero-cost-native (deliverable 4):** wrapper cached compat query **8.7 µs median / 9.6–13.5 µs p99** over two runs — within run-to-run noise of raw Flecs (8.7 µs / 12.2 µs p99). The `&mut dyn FnMut` callback per match is unmeasurable; invariant 4 (hot path) untaxed.
- **Two-backend proof + adversarial paragraph** in the crate-root doc (per-method Loro-projection mapping; the strongest counter-case is *perf, not expressibility* — set-algebra over indexes yields identical matches, only the latency target is Phase-2 work, already tracked in ADR-006). **No-leak enforced in CI:** `ci.yml` greps `core`/`transport`/`plugins` for `flecs_ecs` and fails if found. `/ecs` is the workspace's `unsafe`-forbid exception (own lints: `deny` + pedantic); today it uses zero `unsafe`.
