## 2026-06-20 — describe-to-create resolves local → marketplace for real; the capability web is open-ecosystem-safe

M3.2 made describe-to-create resolve **local-only**; the marketplace + generate tiers were honest stubs.
M5 builds the **marketplace tier** — a queryable index of pre-componentized entries (working objects, not
dead files) — and, because doing that safely forces the capability-identity decision, the
**standard-vocabulary + namespaced-caps** fix that makes the compatibility web safe for an open
ecosystem. ([ADR-015](../decisions/015-capability-identity-and-marketplace-index.md).) Generation +
payments stay seamed — no money moves, nothing generated.

**What landed** (5 green increments on `master`/`origin/main`):

- **Core: namespacing + marketplace + tiered resolve** (`52ec7c8`) — pure metadata, wasm-portable by
  design, no ECS/Loro:
  - `caps.rs`: the identity rule. A bare stdlib name is the `std:` standard vocabulary (`Health` →
    `std:Health`); namespaced names pass through. So the curated stdlib's `"Health"` and a marketplace
    entry's `"std:Health"` intern to the **same** entity, while `acme:Health`/`brandx:Health` stay
    **distinct** — the bare-string collision the M1.3 registry would have had is impossible. (7 tests.)
  - `marketplace.rs`: `MarketplaceEntry` (namespaced provides/requires `CapDecl` with an opt-in
    one-directional `alias_of`, a logical asset name, an inert token price), the `MarketplaceIndex`
    trait (invariant 5 — a remote impl slots in unchanged), and `LocalCatalog` over a checked-in
    builtin dataset. (5 tests.)
  - `resolve.rs`: the real tiered `resolve` — `resolve_local` (offline) **short-circuits**; the
    marketplace index is queried **only** on a no-local-match; nothing anywhere → the generate seam.
    A spy index proves the offline happy path never touches the marketplace. `resolve_local` is
    logic-unchanged. (3 tests.)

- **ECS: capability namespacing + pre-componentized marketplace apply** (`85eb832`): `capscene` interns
  caps by canonical key + records the catalog's `(AliasOf, std:*)` pairs up front; `apply_marketplace_entry`
  instantiates an entry as a working object — its component + namespaced cap pairs (an aliased custom cap
  **also** provides its std cap, so a `std:X` requirer binds it across authors) + the mesh handle — as
  **one undoable transaction**; `persist::Record::ApplyMarketplace` replays it deterministically from the
  catalog by id. **Tests** (`tests/marketplace.rs`, 3): `acme:Health` vs `brandx:Health` do **not**
  cross-bind, yet both `(AliasOf std:Health)` bind a `std:Health` requirer; a no-local-match describe
  resolves the marketplace tier → applies pre-componentized → reveal → bind → undo; the apply survives
  export→replay.

- **Live shell wiring + economy seam** (`6cc6957`): the `describe` command routes local → marketplace →
  generate; a marketplace hit applies pre-componentized with its mesh handle and surfaces the token
  `price` + "creator keeps ~70%" framing as an **inert** UI seam (no purchase). `web/index.html` shows
  the source + price; the generate tier is named, not faked.

- **Tier latencies** (`ae6de6e`; release, 2 runs, i9-13900H): `resolve_local` **p99 ~42 µs** (the
  offline local tier is logic-unchanged — `split_name` only strips a namespace, a no-op for bare stdlib
  caps — well under ADR-012's budget); marketplace query (the **2nd** tier, over the 3-entry local
  catalog) **p99 ~34 µs**. Both far under 16 ms (the marketplace tier needn't beat the frame budget but
  is recorded).

**Adversarial paragraph (the case it breaks):** the marketplace returns a confident-but-wrong entry, or
a malicious author's cap silently hijacks `std:Health`. **Guards:** the `MIN_SCORE` gate (carried from
ADR-012) turns a weak match into an honest no-match — never a wrong one; aliasing is **opt-in** and
**one-directional** toward the standard vocab (an author can only opt *their own* cap into satisfying
`std:Health`, never redefine it), resolved into provides pairs at apply, so no cap can hijack another;
distinct-by-construction namespaced entities make cross-author collision impossible (regression-locked).

**Honest boundaries (named, not stubbed):** no hosted marketplace, payment rail, or creator base — the
index is a checked-in `LocalCatalog` behind the trait a remote index will implement unchanged; the token
ledger / settlement / payout (ADR-004), remote hosting, and text-to-3D generation stay seamed. wasm
parity is **by design** (caps + marketplace are pure metadata) but lives in native-only `/core` — the
full browser *run* awaits the Phase-2 browser backend (same divergence as ADR-012; the `wasm32` tripwire
stays green). The live in-window marketplace demo + dogfood verdict are human-gated.
