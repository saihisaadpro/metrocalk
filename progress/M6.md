## 2026-06-20 — describe-to-create is complete: local → marketplace → generate, with AI a guest

Prompts 23–24 made resolution `local → marketplace`. M6 adds the last tier — **generation** — invoked
only when nothing local or on the marketplace fits: offer "Generate?", drop a **grey placeholder**
instantly, run text-to-3D behind our trait, and **stream the real mesh in** to replace the placeholder.
"AI is a guest, never the foundation": never on the offline path, always last, always metered, and every
AI output enters as a **schema-validated transactional patch**. ([ADR-017](../decisions/017-generation-tier-and-ai-patch-contract.md).)

**What landed** (6 green increments on `master`/`origin/main`):

- **The AI seam** (`94a8352`) — `ai.rs`: an `AiPatch` is an allow-listed op set (`SetField`), **not**
  arbitrary RFC-6902. `apply_ai_patch` validates each op against the registry schema + engine state
  (entity exists · known component · known field · **strict** value type) and applies the whole patch
  through the one commit pipeline as a single undoable tx; any invalid op rejects the **whole** patch
  (all-or-nothing, rejection-as-UX). No raw LLM mutation path. The AI-**edit** sibling rides the same
  path. **6 tests.**
- **The provider boundary** (`0fc0619`) — `generate.rs`: `MeshGenerator` (invariant 5 — returns glb
  **bytes** the caller imports via prompt-23; no SDK types leak). `FakeGenerator` (deterministic +
  offline, returns checked-in bytes after a simulated latency → the loop is CI-testable + the asset is
  content-addressed/replayable); `RemoteGenerator` = the real-provider seam (unconfigured ⇒ unavailable).
  `TokenMeter`/`StubMeter` seam (Generate ≈ 10, Edit ≈ 2; always allows + logs; **no money**). **4 tests.**
- **Placeholder-first + stream-in** (`f7e5dc7`) — `capscene::place_generation_placeholder` (a grey cube
  = empty `MeshRenderer`, provides Renderable + requires Spatial, bindable at once; one undoable tx); the
  real mesh streams in as a **validated AI patch** (same id, swapped handle); undo peels the swap then the
  placeholder. `persist::Record::Generate` → replayed by re-placing + re-applying the stored handle.
  **4 tests:** tier-order (local/marketplace never generate) · the full placeholder→fake-gen→prompt-23-
  import→stream-in→reveal→undo loop · export→replay survives · offline degrades + the offline path is
  unaffected + the placeholder stands alone.
- **Live shell flow** (`f0fbf81`) — `main.rs`: a `generate(query)` command — meter charge → a placeholder
  commits instantly + echoes → the FakeGenerator runs on a **worker thread** (off the hot path, 700 ms
  simulated round-trip) → `GenerateComplete` imports the bytes (adding a *novel* asset to the render store
  at runtime; the fake's prop is idempotent) + streams the mesh in as a validated patch + persists + echoes
  the targeted delta. `web/index.html`: an **opt-in** "Generate?" button on the no-anywhere-match seam;
  offline ⇒ honest seam.
- **Latency** (`f7e5dc7`; release, 2 runs, i9-13900H): `place_generation_placeholder` (the *interactive*
  part — the instant grey placeholder) **p50 ~0.31 ms / p99 ~0.8–1.2 ms @5k**. The generation round-trip is
  provider-latency-bound (network) — **not measured, by design.**

**Adversarial paragraph (the case it breaks):** generation returns confident garbage · a slow/failed
provider hangs the editor · a generated mesh is malicious/huge. **Guards:** the grey placeholder is real +
usable regardless of whether generation ever returns; generation runs on a worker thread (never blocks the
hot path) and an `Err`/unavailable provider just leaves the honest seam; a generated mesh passes the
prompt-23 import validation + size/element limits (64 MiB / 8M, fail-fast) — an honest failure beats a fake
asset; every applied output is a schema-validated patch (over-reaching → rejected).

**Honest boundaries (named, not stubbed):** the **real text-to-3D provider** (config/API key behind the
`RemoteGenerator` seam), the **token ledger/settlement/payout** (prompt 26 — the stub records/logs only),
**remote hosting** (plan §6), the **MCP server** (the patch *contract* is built; the server is a seam), and
**persisting a novel real-provider asset's bytes** across reload (the deterministic fake re-resolves; a
real asset would save its bytes — a documented follow-up). wasm parity by design (the placeholder/apply +
AI-patch are native bridge logic; the provider call is network); the `wasm32` tripwire is unaffected.
