## 2026-06-15 — M3.1 reveal engine landed (headless; live surfacing + dogfood pending)

**What landed.** The deterministic, offline **reveal → rank → explain** engine — north-star #1's
differentiator — built on the confirmed M1.5 `World` compat query + M1.3 registry, headless-tested and
latency-measured. `editor-shell/src/reveal.rs`, on `master` (`c7a1361`), green. ADR-011.

- **Reveal** (`reveal`): per the selection's required caps, the exact M1.5 indexed query
  (`with(Provides, C)` + `without(BindsTo, *)`) → compatible providers-not-yet-bound.
- **Rank** (ADR-011): proximity (Transform distance) → affinity (required-caps provided, via query-
  membership tally) → recency → stable entity-id tiebreak. Deterministic.
- **Explain every "no"** (`why_not`, O(1) per target, on demand): `MissingCapability("Health")` /
  `AlreadyBound` / `NoCapability` — specific, registry-derived, not generic.
- **One-click bind**: reuses the M2.6 `apply_edit` (Bind → `Op::AddBinding` → commit → undoable),
  already tested.

**Measured (release, 5k capability-bearing scene):** reveal (compat-query + rank) **p50 0.706 ms /
p99 1.107 ms** — well under the 16 ms budget. 4 tests: ranked-nearest-first · every-no-explained ·
deterministic · the 5k latency proof. clippy `-D warnings` + fmt clean.

**Honest finding (kept, not hidden):** two earlier drafts blew the budget at 5k — an eager
all-entities `why_not` scan (~33 ms) and a per-match `targets()` affinity read (~25 ms). The committed
design avoids both (indexed query + membership tally + on-demand `why_not`); that's why reveal is
scene-size-independent. Measured in **release** (debug was ~7–26 ms — not the metric, per the
benchmark discipline).
