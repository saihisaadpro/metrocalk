## 2026-06-14 — M2.3 shell-composition gate: **single-window PASS** (M2.1 1b was a GDI artifact)

**What happened.** Built `spikes/shell-composite` (throwaway, seeded from M2.1's `tauri-shell`) and
resolved how the editor shell composites the React UI over the native wgpu viewport on Windows →
[ADR-008](../decisions/008-shell-composition.md). **Verdict: single-window is the shell** (transparent
WebView2 over a native wgpu surface on one HWND). M2.1's 1b "compositing FAIL" was a **GDI capture
artifact**, now disproven.

**Why M2.1 was wrong, and how this held a higher bar.** M2.1 captured 1b with GDI `CopyFromScreen`,
which **cannot see a GPU flip-model/overlay swapchain** → the live wgpu layer read as all-black. This
spike captured with a **DXGI Desktop Duplication** tool (`capture/`) that grabs the DWM's final
composited output (overlays included). Under it the composite is clean; the M2.1 "16×16 collapse" never
reproduced (window stable 1016×739 over 12 s).

Deliverables: (1) **input-routing layer** built first (`input-routing/`, dependency-free, **7 unit
tests green**) — path-agnostic per-pixel UI/viewport hit test, carries to any shell; (2) single-window
confirmed with the **real React panels** (toolbar + inspector + transparent viewport + overlapping
floating box + input log); (3) **fallback ladder not exercised** (single-window passed, so no DComp /
no CEF — ~170 MB avoided); (4) ADR-008 written, ADR-007 reconciliation recorded.

**Decision.** ADR-008 = single-window. No CEF/DComp. ADR-007's compositing-FAIL consequence withdrawn.

**Honest boundaries / blockers.**
- Capture was automated swapchain-aware Desktop Duplication + frame inspection, not a human watching
  an uninterrupted 60 s session; clean motion frames sampled over ~10 s stretches (window-minimize
  hygiene interrupted longer runs) — black stayed 0%, viewport varied every clean frame (no flicker).
- **DPI 100%↔200% monitor move** and a **truly low-end machine** not covered — the two adversarial
  cases held open. The iGPU (the driver-sensitive case M2.1 feared) **was** covered and passed.
- **ADR-007 lives on the unmerged `m2.1` branch**, so its status-line note ("1b was a GDI capture
  artifact — see ADR-008") can't be edited from this lane; apply it at the m2.1↔m2.3 merge.
