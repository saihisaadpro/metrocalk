## 2026-06-14 — M2.5 editor UI scaffold (projection of the core)

**What landed.** `/editor` — the editor front-end as a **projection of the authoritative core**
(invariant 1), delta-fed over the M2.4 transport. Owns `/editor` (not a cargo member). React 19 +
TypeScript + Vite; Zustand · JSON Forms · React Flow · Vitest. ADR-010.

- **Projection store** (Zustand/`useSyncExternalStore`): entity-keyed, immutable per-entity, with a
  separate `{id,name,parentId}` **summary projection** for the tree. `displayed = base ⊕ pending`;
  `displayed[id]` stays ref-identical to `base[id]` when no pending op touches it.
- **Transport client**: TS mirror of the M2.4 Loro-Protocol-v1 envelope (`frame.ts`) + `DeltaClient`
  with separate `%LOR` committed-delta and `%EPH` ephemeral paths; in-process binding + desktop stub
  (M2.6 → Tauri Channel). `MockCore` stands in for the WASM core (swapped M2.6).
- **Inspector**: JSON Forms (over RJSF) with custom renderers via testers (color, `entity-ref`
  bind-target, enum); a field edit → optimistic update + a **JSON-Patch transaction** (AI layer's
  language).
- **Binding graph**: React Flow, neighborhood-scoped (Sigma.js noted for 50k+). **Hierarchy**:
  virtualized over the summary projection (5k). **Reconciliation**: optimistic echo; **rejection is
  first-class UX** ("every 'no' explained"). **Input ownership**: chrome-only for React; viewport
  events left for the native layer (invariant 4), wired in M2.6.

**Verification — 11 Vitest tests green:**
- **Selective subscription at 5k**: editing one entity re-renders exactly its subscribed detail and
  **0 of 5000** hierarchy rows; a name change re-renders exactly that one row.
- **Tear-free** under a React 19 concurrent transition (atomic multi-field delta never half-applied).
- **Reject path**: incompatible optimistic bind reverted + the core's reason surfaced; compatible bind
  confirmed. Optimistic field edit reconciles to a no-op on confirm.
- Envelope round-trip (TS mirror matches M2.4 frame.rs); app end-to-end wiring (5k scene projected
  through client→store→panels, inspector renders on select).

**Numbers.** Single-entity field-edit apply+render ≈ **24–70 ms at 5k in jsdom** (varies with machine
load). The apply is an O(n) shallow map copy (≈ tens of ms at 5k); only the **1** subscribed component
re-renders (0 of 5000 rows). The selective-subscription property is exact; jsdom is slower than a real
browser. A structural-sharing store is the noted upgrade (ADR-010) if browser profiling demands it.
(Fixed an O(n²) bulk-load bug found by the app test — `applyDelta` now copies each map once per delta.)

**Decision.** ADR-010 (editor projection architecture: Zustand-as-projection · JSON Forms over RJSF ·
React Flow neighborhood · optimistic echo with rejection-as-UX · JSON-Patch edit language).

**Honest boundaries.** Viewport hot-input → native layer is **stubbed** (wired in M2.6); `MockCore` →
real WASM/Rust core in M2.6; no real assets; AI/describe-to-create + Rules UI deferred (they reuse the
JSON-Patch `EditTx`). Responsiveness measured in **jsdom**, not a real browser (the selective-render
property is exact either way). The O(n) per-delta map copy is acceptable for the milestone; flagged for
a structural-sharing upgrade.

**Next.** M2.6 convergence: mount this UI in the M2.3 single-window shell, point the desktop transport
binding at the real Tauri Channel (M2.4) + swap `MockCore` for the WASM core, wire the viewport input
hand-off, and integrate the M2.2 render verdict.
