## 2026-06-14 — M2.4 transport protocol + delta wire

**What landed.** The deltas-only transport layer (invariant 2): the wire protocol, the
backend-agnostic `DeltaTransport` trait, three impls, and the `/core` producer hook that turns each
committed transaction into a coalesced delta. Owns `/transport` + `core/src/producer.rs`.

**Design (ADR-009).** Adopted **Loro Syncing Protocol v1 framing** carrying **Loro's own
`export(update, from)` bytes as the opaque payload** — no bespoke per-op codec, no bare-Loro-without-
envelope. Causality (ordering/idempotency/out-of-order) comes from Loro's version vectors for free;
the same wire serves Phase-2 collab unchanged.

- **`/transport` links no Loro** — payloads are `&[u8]`; the Loro `export`/`import` is confined to
  the `/core` producer hook. CI `loro`-outside-`/core` grep simulated green.
- **Envelope:** 16-byte header — magic `frame_kind` (`%LOR` doc-delta · `%ACK` · `%HSK` handshake ·
  `%FRG` fragment · `%EPH` ephemeral **reserved** for Phase-2 · `%PNG`/`%PNT`) · `proto_version` ·
  room id · payload-len. `DocUpdate` = batch id + N length-prefixed update blobs.
- **Shared `DeltaSession` (written once above the byte trait):** frame-tick **coalescing** via
  version-vector diff (`updates_since(last_acked_vv)` → one blob/tick); batch-id/**ACK**;
  **outbox-collapse backpressure** (≤1 DocUpdate in flight; the gap accumulates and the next send is
  ONE spanning update, never N queued); **fragmentation/reassembly** (>256 KiB, for initial
  catch-up); a **reconciliation hook** (Wait-for-Ack, instant-ack in M2, drop-in server-reconcile
  later — kept strictly separate from the engine-side *user* undo stack, M1.6 / ADR-002 F2).
- **3 impls:** in-process (zero-copy sync loopback; browser is Loro-authoritative) · Tauri Channel
  (raw bytes, NOT JSON, over `Channel<InvokeResponseBody>` — sink injected by the shell at M2.6) ·
  local WebSocket (`ws` feature, `tungstenite`, no-TLS loopback; fallback + collab foundation).
- **`/core` producer hook:** `DeltaSource`/`DeltaSink` on `Engine` (export/import+merge); `ProducerHook`
  with `on_tick`/`pump`; **peer identity** (engine's Loro PeerID, ADR-002 F3) established in the
  handshake before any delta flows.

**Verification — 15 tests green.**
- `/transport` (10): envelope round-trip + error cases; fragment reassembly **in-order + out-of-order**
  + timeout sweep; **backpressure-collapse** (stall → 1 frame; ack → 1 spanning update, not N);
  ACK clears the unacknowledged marker; **`%EPH` reserved** (round-trips, ignored, not acked); ping→pong;
  in-process send→apply→ack.
- `/core` (5, two real Loro engines): producer-hook end-to-end convergence (5 commits → 1 coalesced
  frame, both ends learn PeerID); **out-of-order** import (apply update2 before update1 → pending →
  converges); **idempotent** re-import (no-op); **reconnect resync** from a stale vv (one update spans
  the gap); optimistic-echo **no-op** (`repaired == false`).

**Numbers (Win11 26200 · i9-13900H · rustc 1.92.0 · 2 runs each).**

| path | p50 | p99 | note |
|---|---|---|---|
| Channel envelope encode+decode (256 B coalesced delta) | 0.7 µs | 0.9–1.3 µs | ≈0.001 ms — negligible vs 16.6 ms |
| WebSocket loopback round-trip (256 B) | 1.5–3.4 ms | 2.0–3.8 ms | poll-loop-bound (1 ms reader sleep), an upper bound, NOT the wire |

**Decision.** ADR-009 (transport protocol = Loro-Protocol-v1 framing + opaque Loro-update payload).

**Honest boundaries / blockers.**
- **Tauri Channel e2e + Windows snapshot-load deferred to M2.6** — both need the real Tauri shell,
  which is on the unmerged `m2.1` branch. The Channel impl + envelope micro-bench stand alone now; the

- The WS latency is dominated by the foundation's polling reader (1 ms sleep) and swings run-to-run;

- No real collab server / concurrent remote ops yet (Phase 2) — single-process, instant-ack.

## 2026-06-14 — M2.2 real-scene render gate: **PASS**

**What happened.** Built `spikes/render-scene` (throwaway, workspace-excluded like the M0 spikes):
the M1.4 stress scene rendered through one wgpu 29.0.3 crate on native + browser, GPU frame time and
CPU submit measured separately. Resolves the architecture open-question "real-scene render cost at
≥5k entities" (M0 spike ③ only proved a *triangle*).

**Approach (the task's verified research findings, all implemented):**
- **Instancing** — entity cubes + per-entity 3-axis gizmos are each ONE instanced draw; the ground
  grid is a **render bundle** (stable content). Draw-call count is constant (3) by construction.
- **GPU frustum culling → indirect** — a compute pass tests each instance's bounding sphere vs the 6
  frustum planes (Gribb–Hartmann from view-proj), compacts survivors into `visible[]`, and
  `atomicAdd`s the count into a counter copied into the draw-args. Then ONE `draw_indexed_indirect`
  (cubes) + ONE `draw_indirect` (gizmos) — designed **without** multi-draw-indirect and **without**
  `indirect-first-instance` (`first_instance=0`; the vertex shader indexes `visible[]` by
  `instance_index`). Built against `Limits::downlevel_defaults()` (storage-buffer ceiling = 128 MB,
  exactly the browser limit) to prove portability.
- **Timing** — `TIMESTAMP_QUERY` at pass boundaries (portable WebGPU; not the native-only
  `TIMESTAMP_QUERY_INSIDE_ENCODERS`), double-buffered non-blocking readback; CPU submit = wall-clock
  around encode+submit. Scene transforms from the same SplitMix64 seed as M1.4.

**Numbers (full tables → `spikes/render-scene/RESULTS.md`).** RTX 4060 / Vulkan native; Chrome 149 +
Edge 149 (Dawn) browser; 2 runs/discipline.

| discipline | 5k GPU p99 | 20k GPU p99 | budget | result |
|---|---|---|---|---|
| Native (Vulkan) | 0.60 ms | 0.88–0.95 ms | 8.3 / 16.6 ms | ✅ ~14×/17× headroom, no spikes |
| Chrome (Dawn) | 1.34 ms | 3.26 ms | 16.6 ms / 30 fps | ✅ ~750 / ~305 fps-equiv |
| Edge (Dawn) | 1.34 ms | 3.27 ms | 16.6 ms / 30 fps | ✅ |

- **Instancing sanity:** draw calls = **3** at both 5k and 20k; native GPU time grows only **1.4×**
  for a **4×** entity increase (sublinear ⇒ instancing engaged, not a linear non-instanced path).
- **Worst-case frame:** camera path captured ~100% visible (4999/5000, 19988/20000) every run.
- **Storage:** largest buffer 0.15 MB @5k / **0.61 MB @20k** ≪ 128 MB ceiling.
- **Run-to-run swing:** GPU p99 +1.5% (5k) / +7.6% (20k) — under the 25% investigate-threshold.

**Decision.** ADR-003 status-updated: real-scene render gate (M2.2) PASSED; the "non-bindless web
path from day 1" is confirmed sufficient for the editor scene.

**Blocker / gap (parked, low risk).** Firefox 141 leg **not run** — Firefox is not installed on this
machine (M0's winget copy is gone). Firefox WebGPU is built on the same `wgpu`, and the spike uses
only the portable subset (no multi-draw-indirect, `first_instance=0`, downlevel limits), so risk is
low; a Gecko pass (manual, or a non-CDP driver) is the fast follow-up. Recorded in RESULTS.md §5.

**Next.** M2.1 Tauri WebView2 IPC gate (the remaining M2 exit gate). Then author the M2 build from
both verdicts.
