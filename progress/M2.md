### 2026-06-14 (M2.1 — Tauri/WebView2 exit gate: IPC + compositing → ADR-007)

Risk-first M2 entry (the M0 playbook): resolve ADR-003's deferred Windows-WebView2 gate before the
M2 build commits to a shell. Throwaway spike `spikes/tauri-shell/` (standalone Tauri v2 app, its own
workspace; not a member of the root workspace). Two co-equal sub-gates on **real** WebView2
**149.0.4022.69** (Tauri 2.11.2 / wry 0.55.1, wgpu 29.0.3 Vulkan, RTX 4060 laptop). Evidence +
machine-readable results in `spikes/tauri-shell/evidence/`.

- **Sub-gate 1a — IPC delta wire: PASS.** The wire carries the **real** coalesced-delta encoding
  (Loro update bytes from `core::Engine`, a 60 Hz transform drag through the real commit pipeline),
  raw-binary, over three transports. Two 60 s / 3600-frame runs each:
  - Tauri **Channel** (engine→UI push): p99 **3.4–3.6 ms** RTT, **0 dropped**.
  - local **WebSocket** (CEF-style): p99 **1.3–1.7 ms** RTT, 0–1 dropped.
  - `invoke` raw-binary (UI→engine): p99 6.7–6.8 ms RTT (a **~4 ms fixed per-call floor** — use it
    for episodic commits, not the 60 Hz stream).
  - Real delta = **103 bytes**; bandwidth sweep tops at ~31.8 MB/s. **Overhead-bound, not
    bandwidth-bound** → ADR-003's "~200 ms / 10 MB" worst case never applies to a deltas-only wire.
- **Sub-gate 1b — compositing: FAIL (direct single-window approach).** A transparent WebView2 *does*
  composite on Windows (control with wgpu disabled: the desktop shows through the transparent
  viewport). But attaching a native **wgpu surface to the same window HWND** blacks out and collapses
  the window — the webview UI that renders fine on its own disappears. Reproduces Graphite's exact
  conflict (their issue #2541; Tauri #9220/#11944). Not tested: Tauri `unstable` child-webview /
  DComp layering; a human eyes-on pass; min-spec hardware. (See `evidence/audit.md` for the threats
  to validity — chiefly that GDI can't capture GPU overlays, which the control narrows but doesn't
  fully eliminate.)
- **Decision → [ADR-007](../decisions/007-m2.1-tauri-gate-result.md):** IPC leg of ADR-003 confirmed
  (and its bandwidth framing corrected); the one-window wgpu-under-webview composition does not work
  on Windows → plan the shell around **self-composite** (UI-as-texture in the wgpu pass, Graphite's
  proven path), with a cheap child-webview/DComp follow-up spike before any full CEF pivot. Tauri is
  **not** disqualified (IPC passes, transparency works) — but the viewport↔UI composition must be
  self-managed.
- **Spike runs with one command** per mode: `GATE_MODE=bench|composite|selftest pnpm tauri dev`
  (see `spikes/tauri-shell/README.md`). Built + measured on Windows; `cargo`/`tauri dev` green.
